## 🎯 r2 추가 수정 내용

### HAPAG PL 컨테이너 번호 파싱 오류 수정

**문제:** `2200034659 PL.pdf` (HAPAG PL) 파싱 시 컨테이너 번호 0건 반환

**원인:**
- PyMuPDF가 `HAMU272359-6` 을 하나의 단어로 읽지 않고 두 개로 분리
  - word1: `HAMU` (x=7.4%)
  - word2: `272359-6` (x=11.2%)
- 기존 `CT_RE = re.compile(r"^[A-Z]{4}\d{3,}")` 는 단일 토큰만 매칭 → 둘 다 실패

**수정:** `parsers/document_parser_modular/packing_mixin.py`
- CT_RE 단일 토큰 매칭 실패 시 fallback 로직 추가
  - `FOUR_ALPHA = r'^[A-Z]{4}$'` 로 prefix 단어 탐지 (x=5~12%)
  - `NUM_PART = r'^\d{5,7}-?\d?$'` 로 suffix 단어 탐지 (prefix 오른쪽)
  - 두 단어 합산 후 하이픈 정규화 적용
- 결과: `HAMU272359-6` → `HAMU2723596` ✅
