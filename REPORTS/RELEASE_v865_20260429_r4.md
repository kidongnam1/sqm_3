## 1. ARRIVAL 날짜 충돌 처리 — `frontend/js/sqm-inline.js`

### 문제
DO PDF 파싱 결과의 `arrival_date`가 프론트엔드 날짜 입력창에 반영되지 않았음.  
수동 입력값이 있을 경우 파싱값이 조용히 무시되거나 덮어써지는 UX 결함 존재.

### 수정 내용
DO 파싱 성공 직후 3-case 충돌 처리 로직 삽입:

```
Case A │ ARRIVAL 입력창 비어있음  → DO 파싱값 조용히 자동 채움 + CON RETURN 재계산
Case B │ 입력창 값 == 파싱값     → 아무것도 안 함 (이미 일치)
Case C │ 입력창 값 \!= 파싱값     → 우하단 충돌 토스트 팝업 (15초 자동소멸)
        │                           [✅ 파싱값 적용] [🔒 수동값 유지]
        │                           미선택 시 수동값 유지가 기본
```

### 원칙
- 사용자 몰래 덮어쓰기 없음
- 파싱 로그 패널에 모든 선택 결과 기록
- `onestopCalcFreeTime()` 연동 — ARRIVAL 변경 시 CON RETURN 즉시 재계산
