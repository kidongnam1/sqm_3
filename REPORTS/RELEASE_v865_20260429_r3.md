## 🎯 r3 추가 수정 내용

### ONE DO 선사 자동감지 오류 수정

**문제:** `ONE_DO.pdf` 파싱 시 `parse_do()` 가 선사를 인식 못 해 Gemini fallback → None 반환

**원인:**
- `do_mixin.py` 선사 감지 로직이 `"OCEAN NETWORK"`, `"ONEU"`, `" ONE "` 세 가지 키워드만 검사
- ONE_DO.pdf 텍스트에는 위 키워드가 모두 없음 (한글 사무소명 포함, `ONE부산사무소`)
- 파일명 prefix 검사: `('ONEY',)` 에 `ONE_DO.pdf`가 매칭 안 됨 (`ONE_` 로 시작)

**수정:** `parsers/document_parser_modular/do_mixin.py`
- 선사 감지 조건에 `"ONEYS" in _txt` 추가
- ONE_DO.pdf 본문에 BL 번호 `ONEYSCLG01825300` 가 포함되어 있어 즉시 감지됨
- 결과: `_parse_do_one_coord()` 정상 호출 → 좌표 파싱으로 16개 필드 추출

**MRN 분석:**
- PyMuPDF: `26HDMUK026I` 를 5개 단어로 분리 (`26H`, `D`, `M`, `U`, `K026I`) — y=15.6%
- `_parse_do_one_coord()` 의 `all_text` 정규식 방식으로 연속 문자 재조립
- `26HDM UK026I` (공백 포함) → `mrn` 필드 저장
