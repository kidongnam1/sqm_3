### 🟢 LOW

**L1. Bare `except Exception` in worker is broader than the sync function's narrower `except (OSError, ValueError)`.**
- Lines 139, 146.
- Rationale given in inline comment: *"defensive: start_ollama_server already handles its own"*. This is intentional belt-and-suspenders so a logic bug (e.g., AttributeError, TypeError) in the sync function still results in `on_done(False)` rather than a silent dead thread.
- Caveat: This will also swallow `KeyboardInterrupt`-derived exceptions on some Python builds — but Python 3 separates `BaseException` from `Exception`, so `KeyboardInterrupt` and `SystemExit` correctly propagate and terminate the daemon worker. No real bug; noting for completeness.
- **Verdict:** Acceptable as written. Optional tightening could narrow to `except Exception` in the worker but explicitly re-raise on `BaseException` — already the case implicitly.

**L2. No mechanism to coalesce concurrent calls or surface "start already in progress" to the caller.**
- If a UI fires `start_ollama_server_async` twice in quick succession (e.g., user double-clicks), two daemon threads will each run a ~4s start sequence. The second `subprocess.Popen([cli, "serve"])` will fail (port already bound), `check_ollama_server()` will then succeed on the second call's polls, so the user-visible result is *correct* (`True` from both), but a wasted Popen + log noise occurs.
- Not part of the P2 spec. Mentioning so a future P3 patch can add a module-level `_inflight: Optional[threading.Thread]` guard if desired.
- **Verdict:** Out-of-scope nit, no fix required.

### ℹ️ INFORMATIONAL

**I1. Docstring claim "thread-safe" for `pywebview window.evaluate_js`** — correct in current pywebview (≥3.x) but worth periodic re-verification against pywebview release notes.

**I2. The example in the docstring uses `update_ui(ok)` directly inside `_after`** — the comment correctly notes "if your UI toolkit requires updates on the main thread, marshal" but a careless reader could copy the example as-is. Not a defect; the surrounding text is clear.
