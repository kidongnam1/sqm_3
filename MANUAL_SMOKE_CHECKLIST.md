## 사용 시점 (When to use)

- 사용자 PC 배포 전 최종 점검 단계
- 향후 동일 영역 수정 시 재실행 (재현 가능한 표준 절차)

## 사전 준비 (Pre-conditions)

다음 4가지가 모두 충족되어야 테스트 시작:

1. **패치 적용 완료**: P1, P2, P3 코드가 `main` 브랜치에 반영된 상태
2. **롤백 안전망 확보**: `git tag pre-async-patch-20260506` 태그가 존재
   ```bash
   git tag --list | findstr pre-async-patch-20260506
   ```
3. **환경**: Windows 11, Python 3.11+, 가상환경 활성화 완료
4. **잔여 프로세스 없음**: 작업관리자에서 기존 `python.exe`, `pythonw.exe` 모두 종료 확인

## 체크리스트 본문 (8개 항목)

### 1. 시작 시간 측정

- 📋 **항목 명**: 창 등장 속도 측정 (Cold start latency)
- 🎯 **검증 목적**: P1 패치의 핵심 효과 — 백엔드/AI 초기화 대기 없이 스플래시 창이 즉시 등장하는지 정량 확인 (이전 15.7초 → 목표 <1초)
- 🔧 **실행 방법**:
  1. 작업관리자에서 `python.exe` 모두 종료
  2. 스마트폰 스톱워치 앱 준비 (또는 PowerShell `Measure-Command`)
  3. 명령 프롬프트에서 `cd D:\program\SQM_inventory\SQM_v866_CLEAN` 후 `python main_webview.py` 입력
  4. **엔터키와 동시에** 스톱워치 시작
  5. 검은 스플래시 창이 화면에 보이는 순간 스톱워치 정지
- ✅ **합격 기준**: 측정값이 **1.0초 이하** (목표: 0.5~0.8초). 5회 측정하여 4회 이상 1초 이내면 합격.
- ❌ **불합격 시 대응**:
  - 1초 초과 시 → `sqm_debug.log` 의 `[Patch 1]` ~ webview.start 까지 타임스탬프 확인
  - 3초 초과 시 → 즉시 롤백: `git reset --hard pre-async-patch-20260506`

### 2. 스플래시 화면 확인

- 📋 **항목 명**: 스플래시 UI 시각적 검증
- 🎯 **검증 목적**: P1이 의도한 스플래시 HTML이 정상 렌더링되어 사용자에게 "시스템이 작동 중"임을 즉각 알리는지 확인
- 🔧 **실행 방법**:
  1. `python main_webview.py` 실행
  2. 등장한 창을 약 2초간 시각 관찰
  3. (가능하면 화면 캡처)
- ✅ **합격 기준**: 다음 3가지가 **모두** 보일 것
  - [ ] 검은색(어두운 톤) 배경
  - [ ] 회전하는 스피너 애니메이션 (정지 X)
  - [ ] "광양창고 시스템을 시작하는 중..." 한국어 문구
- ❌ **불합격 시 대응**:
  - 흰 화면/빈 화면 → 스플래시 HTML 경로 또는 인라인 문자열 오타 확인
  - 문구가 깨짐 → 인코딩(`utf-8`) 점검
  - 스피너 정지 → CSS animation 누락 확인
  - 30초 내 복구 안 되면 롤백

### 3. 메인 페이지 자동 전환

- 📋 **항목 명**: 스플래시 → 실제 SQM UI 자동 navigate
- 🎯 **검증 목적**: 백엔드(FastAPI) 준비 완료 시 스플래시가 실제 API URL(`http://127.0.0.1:8765/`) 로 정상 전환되는지 확인
- 🔧 **실행 방법**:
  1. 항목 1·2 통과 후 그대로 대기
  2. 스톱워치를 다시 시작 (스플래시 등장 시점부터)
  3. SQM 메인 화면(상단 메뉴 + 재고 테이블)이 보이는 순간 정지
- ✅ **합격 기준**:
  - [ ] **10초 이내**에 자동 전환됨
  - [ ] 전환 시 흰색 빈 페이지 깜빡임이 1초 이상 지속되지 않음
  - [ ] 콘솔 로그에 `INFO uvicorn.error: Application startup complete.` 출력
- ❌ **불합격 시 대응**:
  - 30초 후에도 스플래시 상태 → API 포트 8765 충돌 확인 (`netstat -ano | findstr 8765`)
  - 전환 실패 → `sqm_debug.log` 에서 `navigate(` 호출 누락 여부 확인
  - 즉시 복구 불가 시 롤백

### 4. 재고 데이터 표시

- 📋 **항목 명**: 메인 테이블 데이터 로딩 검증 (P3 DB 비동기 회귀 점검)
- 🎯 **검증 목적**: P3 패치(DB 비동기 헬퍼)로 인해 데이터 조회가 끊기지 않았는지 확인
- 🔧 **실행 방법**:
  1. 메인 화면 진입 후 좌측/상단 "재고 현황" 또는 "재고 목록" 메뉴 클릭
  2. 테이블 영역 관찰
- ✅ **합격 기준**:
  - [ ] 데이터 행이 **1개 이상** 표시됨
  - [ ] 컬럼 헤더(품목명, 수량, 단위 등)가 정상 출력
  - [ ] 로딩 스피너가 5초 이내 사라짐
  - [ ] 콘솔에 SQL 에러(`OperationalError`, `database is locked` 등) 없음
- ❌ **불합격 시 대응**:
  - "데이터 없음" → DB 파일(`*.db`) 경로 확인, 백업본으로 교체
  - SQL 락 에러 → P3 비동기 헬퍼의 connection 닫힘 누락 가능성. 즉시 롤백
  - 무한 로딩 → 백엔드 `/api/inventory` 엔드포인트 직접 브라우저에서 테스트

### 5. 입고 등록

- 📋 **항목 명**: 입고 다이얼로그 + 저장 + UI 반영
- 🎯 **검증 목적**: 쓰기(write) 트랜잭션이 비동기 환경에서도 일관되게 커밋되는지
- 🔧 **실행 방법**:
  1. 메인 화면에서 "입고" 또는 "입고 등록" 버튼 클릭
  2. 다이얼로그가 등장하면 임시 데이터 입력:
     - 품목명: `테스트입고_20260506`
     - 수량: `10`
     - 비고: `스모크 테스트`
  3. "저장" 또는 "등록" 클릭
  4. 메인 재고 목록에서 해당 품목 검색
- ✅ **합격 기준**:
  - [ ] 다이얼로그가 클릭 후 1초 내 등장
  - [ ] 저장 후 성공 메시지 출력 (또는 다이얼로그 자동 닫힘)
  - [ ] 새로고침 없이도 목록에 `테스트입고_20260506` 행이 보임 (또는 새로고침 후 1회 이내 반영)
- ❌ **불합격 시 대응**:
  - 다이얼로그 미등장 → 프론트엔드 JS 에러 (`F12` 개발자 도구 콘솔 확인)
  - 저장 후 반영 안 됨 → DB 커밋 누락 가능성, 롤백
  - 500 에러 → `sqm_debug.log` 의 `/api/inbound` 호출부 스택트레이스 확인

### 6. 출고 등록

- 📋 **항목 명**: 출고 처리 + 재고 수치 차감 검증
- 🎯 **검증 목적**: 입고와 반대 방향(차감) 트랜잭션의 데이터 정합성 확인
- 🔧 **실행 방법**:
  1. 항목 5에서 등록한 `테스트입고_20260506` (수량 10) 의 현재 재고 수량을 메모
  2. "출고" 버튼 클릭 → 동일 품목 선택 → 수량 `3` 입력 → 저장
  3. 다시 재고 목록에서 해당 품목 확인
- ✅ **합격 기준**:
  - [ ] 출고 다이얼로그 정상 등장
  - [ ] 저장 시 에러 없음
  - [ ] 재고 수량이 정확히 **(이전값 - 3)** 으로 갱신됨 (예: 10 → 7)
  - [ ] 음수 재고가 발생하지 않음 (또는 재고 부족 경고가 정상 동작)
- ❌ **불합격 시 대응**:
  - 수량이 차감 안 됨 → DB write 미반영. 롤백
  - 음수 발생하는데 경고 없음 → 비즈니스 로직 회귀. 롤백 후 검토

  - [ ] Excel 에서 열었을 때 한글 깨짐 없음

### 8. 창 닫기 + 좀비 프로세스 검사

- 📋 **항목 명**: 정상 종료 + 잔여 python.exe 미발생 + 재실행 클린 부팅

  - [ ] 작업관리자에 `python.exe` / `pythonw.exe` **0개**
  - [ ] 재실행 시 좀비 정리 메시지 없이 즉시 항목 1의 시작 시간 달성
  - [ ] 재실행 후 항목 4(데이터 표시)가 다시 정상
- ❌ **불합격 시 대응**:
  - python.exe 잔존 → AI/DB 스레드의 daemon 플래그 또는 종료 훅 누락
  - 임시 조치: `taskkill /F /IM python.exe`
  - 근본 원인 못 잡으면 롤백
