## 🔴 CRITICAL 발견 — JS 브릿지 무한 재귀 (즉시 수정됨)

### 문제 진단 (1차+2차 모두 놓침)

**위치**: `main_webview.py` line 470 영역

**기존 코드**:
```javascript
(function installErrorBridge() {
    function report(payload) { ... }
    window.addEventListener("error", ...);
    window.addEventListener("unhandledrejection", ...);
    var _origErr = console.error;          // ← 여기
    console.error = function() {
        ...
        return _origErr.apply(console, arguments);
    };
})();
```

**버그 시나리오**:
1. 사용자가 SQM 메인 페이지에서 링크 클릭 / SPA 라우팅 / `window.location.reload()` 등 호출
2. PyWebView 가 `loaded` 이벤트 재발화
3. `_phase[0] == "main"` 이므로 `evaluate_js` 가 또 실행됨
4. `var _origErr = console.error;` → 이미 wrap 된 함수를 캐치
5. 새 `console.error = function() { ... return _origErr.apply(...) }` → 재귀
6. **N번 발화 시 N단계 재귀** → console.error 호출 시 StackOverflow + 백엔드에 동일 에러 N회 POST

**영향**: 일상 사용 중 콘솔 에러 1번 → 백엔드 로그 폭발 + 브라우저 응답성 저하

### 즉시 수정 적용

```javascript
(function installErrorBridge() {
    // [3차 수정 2026-05-06] Idempotency guard.
    // SPA 라우팅 / link click / load_url 재호출 등으로 on_loaded 가 여러 번
    // 발화될 때, console.error 래핑이 누적되어 무한 재귀가 발생하는 버그 방지.
    if (window.__SQM_BRIDGE_INSTALLED__) {
        console.log("[SQM] Error bridge already installed — skip");
        return;
    }
    window.__SQM_BRIDGE_INSTALLED__ = true;

    function report(payload) { ... }
    // ... rest unchanged
})();
```

**검증**: 두 번째 발화 시 즉시 `return` → 래핑 누적 없음.

## 🟡 P3 — Python 3.9+ 의존성 검증 필요

### 발견

`ThreadPoolExecutor` 워커 스레드의 `daemon=True` 기본값은 **Python 3.9+ 부터**:
- Python 3.7/3.8: 워커 non-daemon → 인터프리터가 워커 종료 대기 → `atexit.register(shutdown, wait=False)` 무력화
- Python 3.9+: 워커 daemon → atexit 의도대로 작동

### 권장 (코드 수정 안 함, 다음 세션)

```toml
# pyproject.toml 또는 requirements.txt 명시
[project]
requires-python = ">=3.9"
```

또는 `database.py` docstring 에 "Requires Python 3.9+" 명시.

**판정**: 🟡 SQM 환경이 Python 3.14 사용 중이므로 (어제 launcher_crash.log 확인) 실질적으로 안전. 명시만 다음 세션에서 추가 권장.
